# Система оптимального поиска игр

## Описание проекта

**Система оптимального поиска игр** — это приложение для управления базой данных видеоигр с реализацией алгоритма оптимального бинарного дерева поиска (Optimal Binary Search Tree, OBST). Проект предоставляет эффективный поиск и сортировку игр по различным критериям с использованием алгоритма динамического программирования Кнута для построения оптимальных деревьев поиска.

## Основные возможности

- 📚 **База данных из 200+ игр** с подробной информацией (название, издатель, дата выхода, рейтинг Metacritic, жанр, платформа и др.)
- 🌳 **5 деревьев оптимального поиска** для различных критериев сортировки
- 🔍 **Быстрый поиск** по названию и издателю
- 📊 **Статистика деревьев** с метриками производительности
- ➕ **Добавление новых игр** с автоматическим обновлением индексов
- 🎮 **Поддержка русского языка** в интерфейсе

### 3. Настройка базы данных

#### Вариант A: PostgreSQL (рекомендуется)

1. Установите PostgreSQL (если не установлен):
   ```bash
   # Ubuntu/Debian
   sudo apt install postgresql postgresql-contrib

   brew install postgresql
   ```

2. Создайте базу данных и пользователя:
   ```sql
   CREATE DATABASE games_db;
   CREATE USER postgres WITH PASSWORD 'postgres';
   GRANT ALL PRIVILEGES ON DATABASE games_db TO postgres;
   ```

3. Обновите настройки подключения в `src/db_manager.py` (при необходимости):
   ```python
   self.db_config = {
       'dbname': 'games_db',
       'user': 'postgres',
       'password': 'postgres',
       'host': 'localhost',
       'port': '5432'
   }
   ```

#### Вариант B: SQLite (альтернативный)

Для использования SQLite измените `db_manager.py` для работы с SQLite вместо PostgreSQL.

## Алгоритм оптимального поиска

Проект реализует **алгоритм построения оптимального бинарного дерева поиска (OBST)** с оптимизацией Кнута.

### Принцип работы

1. **Взвешенное дерево поиска**: Каждому элементу присваивается вес (частота запросов)
2. **Динамическое программирование**: Вычисление оптимальной структуры дерева
3. **Оптимизация Кнута**: Сокращение времени построения с O(n³) до O(n²)

### Матрицы алгоритма

- **AW[i][j]**: Сумма весов элементов от i до j
- **AP[i][j]**: Минимальная стоимость поиска для диапазона [i, j]
- **AR[i][j]**: Индекс корня оптимального дерева для диапазона [i, j]

### Формула стоимости

```
AP[i][j] = AW[i][j] + min(AP[i][k-1] + AP[k][j]) для всех k ∈ [i+1, j]
```

Где:
- `AW[i][j]` — сумма весов в диапазоне
- `AP[i][k-1]` — стоимость левого поддерева
- `AP[k][j]` — стоимость правого поддерева

### Преимущества OBST

- **Среднее время поиска**: O(log n) для сбалансированных деревьев
- **Оптимальность**: Минимальная средняя стоимость поиска среди всех возможных деревьев
- **Адаптивность**: Возможность учёта частоты запросов через веса

## Структура базы данных

### Таблица `games`

| Поле | Тип | Описание |
|------|-----|----------|
| id | SERIAL PRIMARY KEY | Уникальный идентификатор |
| title | VARCHAR(100) | Название игры |
| developer | VARCHAR(100) | Разработчик |
| publisher | VARCHAR(100) | Издатель |
| release_date | DATE | Дата выхода |
| metacritic_score | INT | Рейтинг Metacritic (0-100) |
| genre | VARCHAR(100) | Жанр |
| platform | VARCHAR(255) | Платформы |
| game_modes | VARCHAR(100) | Режимы игры |
| engine | VARCHAR(100) | Игровой движок |
| russian_language | BOOLEAN | Наличие русского языка |
| age_rating | INT | Возрастной рейтинг |

### Критерии сортировки

1. **По названию** (`title`)
2. **По издателю + названию** (`publisher`, `title`)
3. **По дате выхода + названию** (`release_date`, `title`)
4. **По оценке Metacritic + названию** (`metacritic_score DESC`, `title`)
5. **По возрастному рейтингу + названию** (`age_rating`, `title`)

## Производительность

### Метрики

| Метрика | Значение |
|---------|----------|
| Количество игр | 200 |
| Время построения индексов | ~0.5-2 сек |
| Время поиска по названию | <0.01 сек |
| Средняя высота дерева | 8-12 |
| Средневзвешенная высота | 6-8 |

### Факторы производительности

- **Размер базы данных**: Время построения растёт как O(n²)
- **Распределение весов**: Равномерные веса дают сбалансированные деревья
- **Память**: Требуется O(n²) для матриц динамического программирования

## Расширение функциональности

### Добавление нового критерия сортировки

1. Откройте `src/game_index.py`
2. Добавьте новое дерево в метод `build_all_indexes()`:

```python
# 6. По жанру + названию
self.trees['genre_title'] = OptimalSearchTree(
    self.games,
    key_func=lambda x: (x['genre'].lower(), x['title'].lower()),
    weight_func=lambda x: 1
)
self.trees['genre_title'].build()
```

3. Добавьте пункт меню в `src/main.py`
4. Обработайте выбор в главном цикле

### Изменение весов элементов

Для учёта популярности игр измените `weight_func`:

```python
# Вес пропорционален рейтингу Metacritic
weight_func=lambda x: x['metacritic_score']

# Вес на основе даты (новые игры популярнее)
weight_func=lambda x: 100 + (2024 - x['release_date'].year) * 10
```

## Известные ограничения

1. **Память**: Алгоритм требует O(n²) памяти для матриц
2. **Время построения**: Построение дерева для 1000+ игр может занять несколько секунд
3. **PostgreSQL зависимость**: Требуется установленный PostgreSQL сервер
4. **Кодировка**: Поддержка UTF-8 обязательна для кириллических символов
