    # Архитектура системы

    ## Обзор архитектуры

    Система построена по **модульному принципу** с чётким разделением ответственности между компонентами.

    ```
    ┌─────────────────────────────────────────────────────────────────┐
    │                        CLI Interface                            │
    │                         (main.py)                               │
    └─────────────────────────────────────────────────────────────────┘
                                │
                                ▼
    ┌─────────────────────────────────────────────────────────────────┐
    │                     Business Logic Layer                        │
    │                  (game_index.py, optimal_tree.py)               │
    │  ┌─────────────────────┐         ┌─────────────────────────┐   │
    │  │    GameIndex        │         │  OptimalSearchTree      │   │
    │  │  - индексация       │         │  - построение дерева    │   │
    │  │  - поиск            │         │  - поиск по дереву      │   │
    │  │  - статистика       │         │  - статистика           │   │
    │  └─────────────────────┘         └─────────────────────────┘   │
    └─────────────────────────────────────────────────────────────────┘
                                │
                                ▼
    ┌─────────────────────────────────────────────────────────────────┐
    │                      Data Access Layer                          │
    │                      (db_manager.py)                            │
    │  ┌─────────────────────────────────────────────────────────┐   │
    │  │              DatabaseManager                            │   │
    │  │  - подключение к БД                                     │   │
    │  │  - CRUD операции                                        │   │
    │  │  - инициализация схемы                                  │   │
    │  └─────────────────────────────────────────────────────────┘   │
    └─────────────────────────────────────────────────────────────────┘
                                │
                                ▼
    ┌─────────────────────────────────────────────────────────────────┐
    │                       Database Layer                            │
    │                    PostgreSQL / SQLite                          │
    │  ┌─────────────────────────────────────────────────────────┐   │
    │  │  Table: games                                           │   │
    │  │  - id, title, developer, publisher, ...                │   │
    │  └─────────────────────────────────────────────────────────┘   │
    ```

    ## Компоненты системы

    ### 1. CLI Interface (`main.py`)

    **Ответственность**: Взаимодействие с пользователем, обработка команд ввода-вывода.

    **Функции**:
    - Отображение меню
    - Обработка пользовательского ввода
    - Форматирование вывода данных
    - Измерение времени операций

    **Зависимости**:
    - `DatabaseManager` — для работы с данными
    - `GameIndex` — для поиска и сортировки

    **Пример использования**:
    ```python
    db = DatabaseManager()
    db.connect()
    games = db.get_all_games()
    index = GameIndex(games)
    results = index.search_by_title("Witcher")
    print_games(results)
    ```

    ---

    ### 2. Game Index (`game_index.py`)

    **Ответственность**: Управление индексами поиска, координация работы с деревьями.

    **Функции**:
    - Построение 5 деревьев оптимального поиска
    - Предоставление методов поиска по различным критериям
    - Сбор и предоставление статистики

    **Структура**:
    ```python
    class GameIndex:
        trees: Dict[str, OptimalSearchTree]
        
        def build_all_indexes(self):
            # Создаёт 5 деревьев:
            # - title
            # - publisher_title
            # - release_title
            # - metacritic_title
            # - age_title
        
        def get_sorted_by(sort_type: str) -> List[Dict]
        def search_by_title(title: str) -> List[Dict]
        def search_by_publisher(publisher: str) -> List[Dict]
        def get_statistics() -> Dict[str, Dict]
    ```

    **Зависимости**:
    - `OptimalSearchTree` — реализация алгоритма OBST

    ---

    ### 3. Optimal Search Tree (`optimal_tree.py`)

    **Ответственность**: Реализация алгоритма оптимального бинарного дерева поиска.

    **Ключевые классы**:

    #### TreeNode
    ```python
    class TreeNode:
        key: Any           # Ключ для сравнения
        data: Dict         # Данные игры
        weight: int        # Вес узла
        left: TreeNode     # Левый потомок
        right: TreeNode    # Правый потомок
    ```

    #### OptimalSearchTree
    ```python
    class OptimalSearchTree:
        # Матрицы динамического программирования
        AW: np.ndarray     # Суммы весов
        AP: np.ndarray     # Минимальные стоимости
        AR: np.ndarray     # Индексы корней
        
        def build()                    # Построение дерева
        def search(key) -> Dict        # Точный поиск
        def search_partial(str) -> List  # Частичный поиск
        def inorder_traversal() -> List  # Сортированный вывод
        def get_statistics() -> Dict   # Метрики дерева
    ```

    **Алгоритм построения**:

    1. **Сортировка элементов** по ключу
    2. **Вычисление матрицы весов** AW[i][j]
    3. **Заполнение матриц** AP и AR методом динамического программирования
    4. **Построение дерева** из матрицы корней AR

    **Временная сложность**:
    - Построение: O(n²)
    - Поиск: O(log n) в среднем
    - Память: O(n²)

    ---

    ### 4. Database Manager (`db_manager.py`)

    **Ответственность**: Управление подключением к БД, выполнение CRUD операций.

    **Функции**:
    - Подключение/отключение от БД
    - Инициализация схемы из SQL файла
    - Получение данных (все игры, сортированные списки, поиск)
    - Добавление новых записей

    **Конфигурация подключения**:
    ```python
    db_config = {
        'dbname': 'games_db',
        'user': 'postgres',
        'password': 'postgres',
        'host': 'localhost',
        'port': '5432'
    }
    ```

    **Методы**:
    ```python
    def connect() -> bool
    def disconnect()
    def init_database() -> bool
    def get_all_games() -> List[Dict]
    def get_games_sorted_by_*(...) -> List[Dict]
    def search_by_title(title: str) -> List[Dict]
    def search_by_publisher(publisher: str) -> List[Dict]
    def add_game(...) -> bool
    ```

    ## Поток данных

    ### Инициализация системы

    ```
    1. main.py создаёт DatabaseManager
    2. Подключение к PostgreSQL
    3. Инициализация БД из init_db.sql
    4. Загрузка всех игр из БД
    5. Создание GameIndex с загруженными данными
    6. Построение 5 деревьев оптимального поиска
    7. Вывод меню пользователю
    ```

    ### Поиск игры

    ```
    1. Пользователь вводит поисковый запрос
    2. main.py вызывает index.search_by_title(query)
    3. GameIndex делегирует trees['title'].search_partial(query)
    4. OptimalSearchTree обходит дерево, находя совпадения
    5. Результаты возвращаются через цепочку вызовов
    6. main.py форматирует и выводит результаты
    ```

    ### Добавление игры

    ```
    1. Пользователь вводит данные новой игры
    2. main.py вызывает db.add_game(...)
    3. DatabaseManager выполняет INSERT в БД
    4. После успеха main.py перезагружает все игры
    5. Создаётся новый GameIndex с обновлёнными данными
    6. Перестраиваются все деревья поиска
    ```

    ## Принципы проектирования

    ### 1. Разделение ответственности (Separation of Concerns)

    Каждый модуль отвечает за одну область:
    - `main.py` — только UI
    - `db_manager.py` — только работа с БД
    - `game_index.py` — только бизнес-логика поиска
    - `optimal_tree.py` — только алгоритм OBST

    ### 2. Инкапсуляция

    Внутренняя реализация алгоритма скрыта:
    - Матрицы AW, AP, AR приватны
    - Пользователь работает с методами высокого уровня
    - Детали построения дерева инкапсулированы

    ### 3. Единственная ответственность (SRP)

    Каждый класс имеет одну причину для изменения:
    - `TreeNode` изменяется только при изменении структуры узла
    - `OptimalSearchTree` изменяется только при изменении алгоритма
    - `DatabaseManager` изменяется только при изменении схемы БД

    ### 4. Зависимость от абстракций

    `GameIndex` работает с абстракцией дерева, не завися от конкретной реализации OBST.

    ## Масштабируемость

    ### Горизонтальное масштабирование

    - **Кэширование**: Результаты поиска можно кэшировать в Redis
    - **Асинхронность**: Поиск можно выполнять асинхронно
    - **Индексы БД**: PostgreSQL индексы ускорят первичную сортировку

    ### Вертикальное масштабирование

    - **Оптимизация памяти**: Разреженные матрицы для больших n
    - **Параллелизм**: Параллельное построение деревьев для разных критериев
    - **Ленивая загрузка**: Построение деревьев по требованию

    ## Безопасность

    ### Защита от SQL-инъекций

    Используются параметризованные запросы:
    ```python
    self.cursor.execute(query, (f'%{title}%',))
    ```

    ### Валидация ввода

    - Числовые поля проверяются на корректность
    - Даты проверяются на формат
    - Строковые поля ограничиваются по длине

    ## Тестирование

    ### Уровни тестирования

    1. **Модульные тесты**: Тестирование отдельных классов
    2. **Интеграционные тесты**: Взаимодействие компонентов
    3. **Системные тесты**: End-to-end сценарии

    ### Критические точки тестирования

    - Корректность алгоритма OBST
    - Обработка граничных условий (пустая БД, 1 элемент)
    - Обработка ошибок БД
    - Корректность поиска с частичным совпадением
